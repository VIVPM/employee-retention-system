CLUSTER ANALYSIS EXPLANATION
============================

What K-Means did:
It looked at all 15,000 employees' numbers (hours, projects, satisfaction, etc.)
and found 3 natural groups where employees within each group are similar to each other.


THE 3 GROUPS IT FOUND:
----------------------

                    Cluster 0           Cluster 1           Cluster 2
Who                 Low-effort          Overworked          Balanced
Hours               147 (low)           259 (very high)     204 (normal)
Projects            3.3 (few)           4.4 (many)          3.8 (moderate)
Evaluation          0.65 (low)          0.78 (high)         0.73 (medium)
Churn               28.9%               33.5%               6.2%


WHY THIS HELPS TRAINING:
-------------------------

The reason employees leave is different in each cluster:

- Cluster 0 employees leave because they're BORED
  -> low hours, few projects, low evaluation. They feel overlooked.

- Cluster 1 employees leave because they're BURNT OUT
  -> 259 hours/month, 4.4 projects, yet similar low satisfaction as Cluster 0.

- Cluster 2 employees RARELY LEAVE (6.2%)
  -> they have a balanced workload.

If you train ONE model for everyone, it tries to learn one rule that explains
all three patterns. That's hard -- "bored" and "burnt out" are opposite reasons
for the same outcome (leaving).

By training one model PER cluster:
- RandomForest0 only learns: "among low-effort employees, who leaves?"
- RandomForest1 only learns: "among overworked employees, who leaves?"
- RandomForest2 only learns: "among balanced employees, who leaves?"

Each model has a simpler, cleaner pattern to learn -- that's why accuracy is higher.


INTERVIEW ONE-LINER:
--------------------
"A single model struggled because employees leave for opposite reasons -- burnout
vs boredom. Clustering first lets each model specialize on one churn pattern."
